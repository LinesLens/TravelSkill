# Travel Skill

通过对话收集用户偏好，生成中文旅行攻略 Markdown 文档 + HTML 移动端查看器，附带 QR 码方便手机扫码查看。

## 核心原则

- 一问一答，每次只问一个问题
- **用户已提供的信息绝不重复询问** — 每条信息只问一次，即在对话中已出现的（如"我想去东京"）直接从问题列表移除
- 信息越多攻略越精准，但允许用户跳过（提供合理默认值）
- 所有输出使用中文
- 攻略数据结构统一，按层级展示
- 行程节奏从兴趣偏好自动推断（"休闲度假"→悠闲，"户外探险"→紧凑，默认适中）
- 雨天备选方案默认包含，无需询问
- **默认用餐时间：早餐 08:30、午餐 11:30、晚餐 18:00**，根据当天行程可微调 ±30 分钟
- **关键时间节点（发车、入园、表演等）必须高亮标注**

## MCP 环境准备阶段

**在询问任何旅行信息之前完成 MCP 环境检查，一次性呈现结果。**

### 第一步：检测已有 MCP

检查系统提供的 function list 中 `mcp__` 前缀的工具，按以下类别归类：

| # | 类别 | 常见 MCP 名称模式 | 用途 |
|---|------|-----------------|------|
| 1 | 地图 | `mcp__google-maps` / `mcp__amap` / `mcp__gaode` | 路线规划、距离计算、交通时间 |
| 2 | 火车票 | `mcp__rail` / `mcp__12306` | 车次查询、票价、余票 |
| 3 | 机票 | `mcp__flight` / `mcp__skyscanner` | 航班时刻、价格对比 |
| 4 | 天气 | `mcp__weather` / `mcp__openweather` / `mcp__qweather` | 目的地天气预报 |
| 5 | 汇率 | `mcp__exchange` / `mcp__currency` | 国际旅行货币换算 |

### 第二步：批量呈现结果

一次性告知用户 MCP 就绪情况和缺失项，不逐个询问。

对缺失的 MCP，简述其价值。用户可输入编号选择安装，或回复"跳过"全部使用搜索降级。

示例：
> "MCP 环境检测：✅ 地图、天气 / ⚠️ 火车票、机票、汇率缺失。安装哪些？（输入编号或'跳过'）"

### 第三步：协助安装

用户选择安装时：
1. `WebSearch` 搜索该 MCP 的最新安装文档和 npm 包名
2. 提供安装命令和配置 JSON 片段
3. 不自动执行安装命令，由用户操作完成后确认

## 信息收集阶段

按以下顺序逐一询问，每次一个问题。**如用户在对话中已提供某项信息，跳过该问题。**

| # | 问题 | 选项 |
|---|------|------|
| 1 | **目的地** | 城市或国家（可多个） |
| 2 | **出行日期** | 具体日期或大致时间段 |
| 3 | **预算** | 经济型 / 舒适型 / 豪华型 / 自定义金额 |
| 4 | **出行人数** | 独自 / 情侣 / 家庭 / 朋友团体，是否有老人或儿童 |
| 5 | **兴趣偏好** | 自然风光 / 历史文化 / 美食 / 购物 / 户外探险 / 休闲度假 / 网红打卡 / 演出赛事 |
| 6 | **交通偏好** | 公共交通 / 租车自驾 / 包车 / 步行友好 |
| 7 | **住宿偏好** | 青旅 / 民宿 / 经济酒店 / 高档酒店 / 度假村 |
| 8 | **饮食要求** | 无特别要求 / 素食 / 清真 / 当地特色优先 / 高档餐厅体验 |
| 9 | **其他需求** | 自由文本（提示：特殊日期、无障碍需求、签证情况等） |

## 攻略生成阶段

### 信息搜索

根据用户偏好，按以下优先级获取各类信息：

**MCP 优先 → WebSearch 降级：**

| 类别 | MCP 优先 | 降级方案 | 标注 |
|------|----------|---------|------|
| 地图/路线 | Google Maps / Amap MCP | WebSearch 估算 | ✅ 实时 / 📌 参考 |
| 火车票 | 12306 / Rail API MCP | WebSearch 参考车次 | 同上 |
| 机票 | Flight / Skyscanner MCP | WebSearch 参考航班 | 同上 |
| 天气 | OpenWeather / 和风天气 MCP | WebSearch 历史趋势 | 同上 |
| 汇率 | Exchange Rate MCP | WebSearch 近期汇率 | 同上 |

搜索内容清单：
- 核心景点、开放时间、门票价格、**关键时间节点**（表演时间、换岗时间、最佳入场时间等）
- 景点间距离和推荐路线（地图 MCP 优先）
- 大交通方案：机票/火车票（Flight/Rail MCP 优先）
- 当地特色美食和推荐餐厅：**具体餐厅名称、地址、大众点评评分（0-5.0）、评论数、人均价格、必点菜品、预订方式**
- 住宿区域推荐：**具体酒店名称、地址、大众点评评分（0-5.0）、评论数、人均价格/晚**
- 当地交通卡/通票信息
- 出行期间天气预报（天气 MCP 优先）
- 汇率预算换算（汇率 MCP 优先，国际旅行场景）
- 签证要求（中国护照）
- 当地习俗和注意事项
- 常用当地语言短语
- 紧急联系方式
- 大众点评/携程评分和评论数（餐厅和酒店必须标注）

### 攻略内容

使用 `${CLAUDE_SKILL_DIR}/references/data-structure.md` 定义的数据结构组织信息。

**以下内容默认包含，无需询问用户：**
- **每日三餐** — 每天行程必须包含早餐（🥣 08:30）、午餐（🥢 11:30）、晚餐（🍖 18:00），以独立的带高亮样式行呈现。用餐时间可根据当天行程微调 ±30 分钟（如赶早班车则早餐提前至 08:00），但必须标注具体时间
- **重大事件节点** — 每天 5-8 个带时间标注的关键节点。三餐和重要时间点（发车、入园、表演、返程等）必须使用彩色时间徽章（`.key-time`）突出显示，普通活动使用标准时间轴样式
- **雨天备选方案** — 每天行程附带 1-2 个室内备选活动
- **交通卡/通票** — 当地交通卡信息及购买建议
- **自由时间** — 每天预留 1-2 小时自由探索
- **地图链接** — 景点间路线使用 Google Maps/Amap 短链接
- **餐厅预订提示** — 热门餐厅是否需要提前预订及方式
- **大众点评数据** — 餐厅和酒店必须标注评分（0-5.0）和评论数

**生成后执行合理性校验：**
- 同一天相邻景点间交通时间是否合理
- 全天活动是否过于紧凑（景点数 ≤ 4，总时长 ≤ 10 小时）
- 发现不合理时自动调整

## 图片搜索阶段

与内容生成并行：确定景点和美食列表后立即启动图片搜索，不等内容写完。

### 方案 1：Pexels API（主方案）

```
https://api.pexels.com/v1/search?query={英文关键词}&per_page=2
```

免费额度每月 20000 请求，无需认证即可满足每次攻略的图片需求。

### 方案 2：Pixabay API（备选）

```
https://pixabay.com/api/?q={英文关键词}&per_page=2
```

Pexels 不可用时使用。

### 方案 3：WebSearch 降级

搜索 `"{景点英文名} photo free image"` 获取可用图片链接。

### 搜索范围

- 每个核心景点 1-2 张
- 每种推荐美食 1 张
- 每个推荐住宿区域 1 张
- 每个拍照打卡点 1 张

## 输出阶段

使用 `${CLAUDE_SKILL_DIR}/references/output-template.md` 中的模板生成两个文件：

1. **`<目的地>-旅游攻略.md`** — Markdown 主文档，适合编辑和版本管理
2. **`<目的地>-旅游攻略.html`** — HTML 移动端查看器，适合手机扫码查看（含 QR 码）

两个文件内容一致，格式不同。

### 校验清单

输出前确认：
- [ ] 每天包含早餐（08:30）、午餐（11:30）、晚餐（18:00），含具体餐厅、人均、评分
- [ ] 每天包含 5-8 个重大事件节点（含三餐 + 关键活动），时间轴形式呈现
- [ ] 餐饮含具体餐厅名、地址、大众点评评分（0-5.0）、评论数
- [ ] 住宿含具体酒店名、人均价格/晚、大众点评评分（0-5.0）、评论数、地址
- [ ] 每天行程包含上午/下午/晚上/自由时间
- [ ] 每天包含雨天备选方案
- [ ] 交通卡/通票信息已包含
- [ ] 地图链接已标注
- [ ] 餐厅有预订提示
- [ ] 预算表覆盖所有类别
- [ ] 每条数据标注来源类型（✅ 实时 / 📌 参考）
- [ ] 图片 URL 可访问
- [ ] `.md` 文档格式完整，表格对齐
- [ ] `.html` 移动端适配（viewport + 响应式 CSS），QR 码可正常生成

## QR 码与手机分享阶段

两个文件生成完成后，执行以下步骤让用户可扫码在手机上查看 HTML 攻略：

### 第一步：启动本地 HTTP 服务

在攻略文件所在目录启动 Python HTTP 服务（后台运行）：

```bash
cd <输出目录> && python3 -m http.server 8899 &
```

### 第二步：获取本机局域网 IP

```bash
hostname -I | awk '{print $1}'
```

### 第三步：生成并展示 QR 码

QR 码指向 HTML 查看器（移动端友好渲染）：

**方案 A — 下载 QR 图片（推荐）：**

```bash
python3 -c "
import urllib.request
url = 'http://<IP>:8899/<目的地>-旅游攻略.html'
api = f'https://api.qrserver.com/v1/create-qr-code/?size=300x300&data={urllib.request.quote(url)}'
urllib.request.urlretrieve(api, '/tmp/travel_qr.png')
print(f'QR 码已保存: /tmp/travel_qr.png')
print(f'手机扫码地址: {url}')
"
```

**方案 B — 终端 ASCII QR 码：**

使用 `curl "qrenco.de/<URL编码后的地址>"` 在终端直接显示 ASCII 二维码。

**方案 C — HTML 内嵌自生成（已默认包含）：**

HTML 模板已内嵌 qrcodejs 库，页面在浏览器中打开后自动在底部生成当前页 URL 的二维码。

### 第四步：引导用户分享

告知用户：
1. 确保手机和电脑连接同一 Wi-Fi
2. 使用手机相机扫描 QR 码图片（或 HTML 页面底部的 QR 码）
3. 手机浏览器打开 HTML 攻略后，可「添加到主屏幕」实现离线查看
4. `.md` 文档可用 Markdown 阅读器或文本编辑器查看编辑
5. 如需停止服务：`pkill -f "python3 -m http.server 8899"`
