# Travel Skill

通过对话收集用户偏好，生成包含图片的中文旅行攻略 Markdown 文件。

## 核心原则

- 一问一答，每次只问一个问题
- 信息越多攻略越精准，但允许用户跳过（提供合理默认值）
- 所有输出使用中文
- 攻略数据结构统一，按层级展示

## MCP 环境准备阶段

**在询问任何旅行信息之前，必须先完成 MCP 环境检查。**

### 第一步：扫描可用 MCP

检查当前环境已安装的 MCP 工具，列出所有可用的工具名称。

### 第二步：对照旅行 MCP 需求表

逐一检查以下五类 MCP 是否已安装：

| # | 类别 | 常见 MCP 名称 | 用途 |
|---|------|--------------|------|
| 1 | 地图 | `@anthropic/mcp-server-google-maps` / `amap-mcp` / `gaode-mcp` | 路线规划、距离计算、交通时间 |
| 2 | 火车票 | `rail-api-mcp` / `12306-mcp` | 车次查询、票价、余票 |
| 3 | 机票 | `flight-api-mcp` / `skyscanner-mcp` | 航班时刻、价格对比 |
| 4 | 天气 | `openweather-mcp` / `qweather-mcp` | 目的地实时天气预报 |
| 5 | 汇率 | `exchange-rate-mcp` | 国际旅行货币换算 |

### 第三步：逐个询问缺失的 MCP

对每个缺失的 MCP，按上述顺序逐一询问用户：

> "我没有检测到 **[MCP 名称]**，它可以在做攻略时提供 **[具体价值]**。要我帮你安装配置吗？"

- 用户同意 → 进入第四步协助安装
- 用户拒绝或跳过 → 标记为该类别使用降级方案（WebSearch），继续下一个
- 所有 MCP 检查完毕后 → 进入信息收集阶段

### 第四步：协助安装 MCP

用户同意安装某个 MCP 时，执行以下操作：

1. 使用 `WebSearch` 搜索该 MCP 的最新安装文档和 npm 包名
2. 向用户说明安装步骤：
   - 用 `npm install -g <mcp-package>` 或 `npx <mcp-package>` 安装
   - 在 Claude Code 的 MCP 配置文件中添加对应配置
3. 确认用户完成安装后，继续下一个缺失 MCP 的询问

**注意：** 不要自动执行安装命令，只提供清晰的安装指引让用户操作。

## 信息收集阶段

依次询问以下信息，一次一个问题：

1. **目的地** — 城市或国家（可多个目的地）
2. **出行日期** — 出发和返回日期，或大致时间段（如"明年春节"）
3. **预算** — 人均预算范围（经济型/舒适型/豪华型，或具体金额）
4. **出行人数** — 独自出行/情侣/家庭/朋友团体，成人及儿童人数
5. **兴趣偏好** — 自然风光/历史文化/美食/购物/户外探险/休闲度假/网红打卡
6. **交通偏好** — 公共交通/租车自驾/包车/步行友好
7. **住宿偏好** — 青旅/民宿/经济酒店/高档酒店/度假村
8. **饮食要求** — 无特别要求/素食/清真/当地特色优先
9. **其他需求** — 自由文本，如"带老人需要无障碍设施"、"想赶早班机"等

如果用户在对话中已提供了部分信息，跳过对应问题。

## 攻略生成阶段

### MCP 工具降级策略

优先使用 MCP 工具获取实时数据，不可用时降级到 WebSearch。每条数据标注来源类型：

| 类别 | 用途 | MCP 优先 | 降级方案 |
|------|------|----------|---------|
| 地图/路线 | 景点距离、交通时间、路线规划 | Google Maps / Amap MCP | WebSearch 估算距离和通勤时间 |
| 火车票 | 车次、票价、余票 | 12306 / Rail API MCP | WebSearch 查询参考车次和票价范围 |
| 机票 | 航班时刻、价格对比 | Flight API / Skyscanner MCP | WebSearch 查询参考航班和价格范围 |
| 天气 | 出行期间实时预报 | OpenWeather / 和风天气 MCP | WebSearch 查询历史天气趋势 |
| 汇率 | 国际旅行预算换算 | Exchange Rate API MCP | WebSearch 查询近期汇率 |

**降级规则：**
- MCP 工具可用时：调用 MCP 获取数据，标注 `✅ 实时数据`
- MCP 不可用时：使用 WebSearch 获取参考数据，标注 `📌 参考数据，建议出行前核实`
- 每次使用前先检查可用 MCP 工具列表，按类别匹配

### 搜索信息来源

在生成攻略内容前，根据 MCP 降级策略获取以下信息：

- 目的地的核心景点、开放时间、门票价格
- 景点间距离和推荐路线（地图 MCP 优先）
- 大交通方案：机票/火车票（Flight/Rail MCP 优先）
- 当地特色美食和推荐餐厅
- 住宿区域推荐
- 当地天气趋势（天气 MCP 优先）
- 汇率预算换算（汇率 MCP 优先，国际旅行时）
- 签证要求（中国护照）
- 当地习俗和注意事项
- 常用当地语言短语
- 紧急联系方式（报警、急救、大使馆）

### 统一数据结构

所有信息存储在统一数据模型中：

```
TravelGuide
├── meta: 目的地、日期、预算、人数、偏好
├── overview: 目的地简介（2-3句）
├── days[]:
│   ├── day_number
│   ├── date
│   ├── morning:   活动描述、景点、预估时间
│   ├── afternoon: 活动描述、景点、预估时间
│   ├── evening:   活动描述、餐厅、预估时间
│   ├── attractions[]: 名称、描述、门票、开放时间
│   ├── food[]: 餐厅名、推荐菜品、人均价格
│   └── transportation: 当天交通建议
├── accommodation[]: 区域推荐、酒店类型、价格区间
├── budget_table: 分类预算估算（交通/住宿/餐饮/门票/购物/其他）
├── weather: 出行期间天气趋势、穿衣建议
├── visa: 签证类型、办理方式、所需材料
├── customs: 当地习俗、禁忌、礼仪
├── language[]: 常用短语（原文+发音+中文）
├── emergency: 报警/急救/大使馆电话和地址
└── photo_spots[]: 最佳拍照地点、时间建议
```

## 图片搜索阶段

对攻略中涉及的以下内容搜索图片：
- 核心景点（每个景点 1-2 张）
- 推荐美食（每种 1 张）
- 住宿区域（每个推荐区域 1 张）
- 打卡拍照点（每个 1 张）

使用 Unsplash Source API 搜索图片：
```
https://source.unsplash.com/featured/?{英文关键词}
```

在 Markdown 中嵌入图片时使用该 URL，Unsplash 会自动返回相关图片。
